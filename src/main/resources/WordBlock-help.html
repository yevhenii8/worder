Command Line Interface

Initially, Worder was a Java CLI application, then a Kotlin CLI application. As for now it's a GUI one :) But utilizing CLI is still the most convenient and a very handy way to choose a resolution for WordBlocks. Some important notes here are:

  • CLI is only connected to the current NO_RESOLUTION block.
  • When first resolution received, you can only change it through GUI.

The intended way of using CLI is the next: you choose DEF (definition(s)), EXP (example(s)) and then CLI sets WordBlock's resolution to UPDATED. You can select from proposed options (using their indexes) or from the custom ones. Also, you can combine proposed and custom in any way.

Possible ways to choose definition(s) - DEF?
  1. Two proposed: (i1 i2) -> (ENTER)
  2. Two custom: (c1) -> (ENTER) -> (c2) -> (ENTER)
  3. Only proposed: (i1) -> (ENTER) -> (ENTER)
  4. Only custom: (c1) -> (ENTER) -> (ENTER)
  5. Proposed and custom: (i1) -> (ENTER) -> (c1) -> (ENTER)
  6. Custom and proposed: (c1) -> (ENTER) -> (i1) -> (ENTER)
Possible ways to choose example(s) - EXP?
  1. No example: (ENTER)
  2. One or more proposed: (i1 i2 i3 ... in) -> (ENTER) -> (ENTER)
  3. One or more custom: (c1) -> (ENTER) -> (c2) -> (ENTER) -> ... -> (ENTER) -> (ENTER)
  4. Proposed and custom: (i1) -> (ENTER) -> (c1) -> (ENTER) -> ... -> (ENTER) -> (ENTER)
  5. Custom and proposed: (c1) -> (ENTER) -> (i1) -> (ENTER) -> ... -> (ENTER) -> (ENTER)
At any moment (DEF or EXP stages) you can set another resolution or ask for help:
  1. --remove -> immediately sets Removed resolution
  2. --learn -> immediately sets Learned resolution
  3. --skip -> immediately sets Skipped resolution
  4. --help -> displays this help message

Update Tab Guide

To update a word means to pick up a non-updated word from your database, request all possible transcriptions, definitions and examples from the present Requesters. Then to compose a WordBlock which allows you to choose definitions and examples for the word and consequently update the word in database.

Which words will Worder pick up for updating?
  1. It doesn't not have worder's #updated Tag.
  2. Its rate < 100, so it's unlearned word.
  3. It was not skipped in current session.
Can I configure pick-up criteria?
As for now - No.
Which actions are available for WordBlock?
  1. Update
  2. Set Removed
  3. Set Learned
  4. Set Skipped
What I need to choose in order to update a word?
  1. You have to choose only one (primary) definition. Secondary is optional.
  2. You can also choose zero or more (unlimited) count of examples.
  3. Transcription is picked up automatically, preferring the present one.
